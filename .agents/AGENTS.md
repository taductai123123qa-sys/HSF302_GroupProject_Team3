# Strict Rules
1. Khi có interface, BẮT BUỘC phải viết luôn cả file triển khai thực tế (Implement - ví dụ: ServiceImpl) một cách hoàn chỉnh, không viết mã giả (pseudo-code) hay để trống comment kiểu "// TODO: implement this".
2. BẮT BUỘC cung cấp một mục "Walkthrough" (Giải thích luồng chạy) ngắn gọn, súc tích bằng tiếng Việt ở cuối câu trả lời để giải thích rõ dữ liệu đi từ đâu đến đâu, xử lý biến đổi như thế nào.
